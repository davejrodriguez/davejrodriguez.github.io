ATK<\/strong> (Audio Tool Kit) is a demo of ATK-Sharp<\/a>, a procedural audio generation library. ATK contains four example sound scripts that developers can use as a reference to script their own custom sounds.<\/p>\n\n\n\n

Download now<\/a> from the Unity Asset Store!<\/p>\n\n\n\n

Why use procedural audio?<\/strong><\/h2>\n\n\n\n

Using procedurally-generated audio instead of pre-made audio files has two main benefits:<\/p>\n\n\n\n

1. Build file size can be reduced.<\/em> Audio files can take up megabytes of disk space, while procedural audio is an order of magnitude smaller, typically only requiring kilobytes to store its script\u2019s algorithm.<\/p>\n\n\n\n

2. Procedural audio is flexible.<\/em> Audio files are essentially baked. You may be able to change the pitch or speed of a file, but they could never match the sample-by-sample control you\u2019ll have using procedural audio. Adding physics and logic as input, a given sound script may never produce the same output more than once.<\/p>\n\n\n\n

Features<\/strong><\/h2>\n\n\n\n

\u2013 WindAudio.cs demonstrates the very basics of what a developer will need to do to start scripting procedural audio using the library.
\u2013 FireAudio.cs script showcases how the developer can combine noise generators and a modifier to create the hiss, crackle, and pop of a campfire.
\u2013 ChimeAudio.cs exhibits how the developer can use physics as an input to the audio script.
\u2013 CricketAudio.cs models the chirp pattern of an actual species of cricket, known as the fall field cricket, to demonstrate how far the concept of procedural audio can be taken.<\/p>\n\n\n\n
